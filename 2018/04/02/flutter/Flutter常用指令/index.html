Flutter常用指令

Flutter 的常用指令主要分为 开发工具命令（flutter 命令行）和 Dart 工具命令（dart 命令行），以下是核心指令整理：

一、Flutter 常用命令

1. 项目创建与运行

  • 创建新项目
  • 运行项目（自动选择设备）
    • 指定设备：flutter run -d <设备ID>（通过 flutter devices 查看设备ID）
    • 生产模式：flutter run --release
    • 调试模式：flutter run --debug
  • 快捷指令
    • r Hot reload.
    • R Hot restart.
    • h List all available interactive commands.
    • d Detach (terminate “flutter run” but leave application running).
    • c Clear the screen
    • q Quit (terminate the application on the device).
      如果是想重新进入调试模式可以使用flutter attach

2. 构建应用

  • 生成 APK（Android）
    • 分架构构建：flutter build apk --target-platform android-arm,android-arm64
  • 生成 App Bundle（Google Play）
  • 生成 IPA（iOS）
    • 需要 Xcode 导出：flutter build ipa

3. 设备与模拟器

  • 查看已连接设备
  • 启动模拟器
  • 清理设备日志

4. 依赖管理

  • 获取依赖
  • 升级依赖
  • 添加依赖（手动修改 pubspec.yaml 后运行 flutter pub get）

5. 测试与调试

  • 运行单元测试
  • 运行集成测试
  • 检查代码格式
  • 格式化代码

6. 热重载与热重启

  • 热重载（Hot Reload）
    运行应用时按 r（保留状态快速更新界面）。
  • 热重启（Hot Restart）
    运行应用时按 R（重启应用并保留状态）。

7. 升级与版本管理

  • 升级 Flutter SDK
  • 切换 Flutter 版本
  • 查看当前版本

二、Dart 常用命令

  • 运行 Dart 脚本
  • 编译为可执行文件
  • 生成 AOT 快照

三、其他实用命令

  • 清理构建缓存
  • 查看 Flutter 环境配置 （检查 Android/iOS 开发环境是否配置正确）
  • 生成图标和启动页
    使用第三方包（如 flutter_launcher_icons）后运行：

常见场景示例

  1. 从零启动项目

  2. 发布到 Google Play

  3. 调试性能问题

掌握这些命令后，Flutter 开发效率会大幅提升！遇到问题时，记得用 flutter doctor 检查环境配置。
